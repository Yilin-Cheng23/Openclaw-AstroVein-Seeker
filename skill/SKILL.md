# 学术人脉深度分析（Academic Mentor Intel）

三个互相关联的能力，生成精美HTML报告，帮助研究生高效管理学术人际关系。

---

## ⚠️ 数据准确性红线（最优先级）

### 绝对不能犯的错误（有记录的教训）
- ❌ [导师姓名] ≠ 少年班出身（已证伪，不要再出现）
- ❌ [师兄姓名] ≠ [导师姓名]导师（[师兄姓名]是[导师姓名]的师兄）
- ❌ [同门A] ≠ [导师姓名]导师（[同门A]是[导师姓名]的师弟）
- ✅ [导师姓名]的导师是：**[导师的导师]**
- ❌ 不要把"三角镇官员[同门A]"当成"研究生[同门A]"（实体消歧！）

### 搜索时主动确认关系
每次出现新的人名关系，必须先搜索核实：
- 某人的导师是谁？→ 查 arXiv 早期论文 acknowledgement 或 thesis affiliation
- 某人和某人是师兄弟还是导师关系？→ 不要假设，要查

### 论文搜索范围要广
- ❌ 不要只在小团队合作圈内搜
- ✅ 要用 arXiv + DBLP + Semantic Scholar 多源搜索
- ✅ 搜索关键词要包括：姓名全拼、常见英文名变体（如 "Ji-Yan He" / "Jiyan He"）
- ✅ DBLP 专门收录 CS 顶会，是 arXiv 的重要补充

---

## 🚫 报告措辞红线

**以下词汇/说法禁止出现在报告中：**
- ❌ "已核实" / "经核实"
- ❌ "交叉验证确认"
- ❌ "经过验证的论文"
- 理由：报告可能出错，说了"核实"反而更尴尬

**正确替代说法：**
- ✅ "来源：arXiv / DBLP"
- ✅ "以下为搜索到的相关论文"
- ✅ "（信息来源：公开论文 affiliation）"
- ✅ "（仅供参考，建议结合本人简历确认）"

## 🎯 三大核心能力

### 能力1：导师向上管理
分析导师的学术画像、最新动态、性格特点，生成与导师沟通的具体策略。

**适用场景：**
- 不了解导师最新关注点，不知道怎么汇报
- 想找合适时机提idea或申请参与项目
- 需要了解导师资源和关系网络

**输出内容（按报告章节顺序）：**
1. 研究演变轨迹（分阶段时间轴）
2. 深度洞察 · 性格与工作风格（研究风格、合作模式、性格推断）
3. 向上管理行动手册（具体沟通建议、话术、时机）
4. 近期动态时间线（6个月，arXiv + 公众号内容摘要）
5. 历史重要成果（5年，不要只找小圈子，广泛搜索）
6. 总结与建议

---

### 能力2：同门近期画像
分析同门的学术背景、研究进展、性格爱好，为合作打基础。

**输出内容（按报告章节顺序）：**
1. 研究演变轨迹（分阶段时间轴）
2. 深度洞察（研究风格、技能图谱、性格推断）
3. 核心合作者网络
4. 如何与他建立联系（具体策略、共同话题切入点）
5. 近期动态时间线（6个月）
6. 历史成果（5年，广泛搜索）

---

### 能力3：合作共赢规划
基于能力1和能力2的分析，生成具体可行的合作idea、分工方案和人际关系策略。

**输出内容（按报告章节顺序）：**
1. 人际关系维护策略（最前面！最吸引人的干货）
2. 合作 Ideas（3-5个具体可行方向，含分工方案）
3. 综合推进策略与时间线

---

## ⚡ 效率优先原则

**并行搜索（必须）：**
- 多人画像：使用 `sessions_spawn` 并行子 agent，每人一个子 agent 同时生成
- 单人搜索：同时发起 arXiv + DBLP + 微信搜一搜 + 百度

**速度目标：**
- 单人报告：3分钟以内
- 两人并行：总时间与单人相当

## 🔍 数据收集规则

### arXiv 论文搜索
- ❌ **绝对不能**只靠姓名搜索就认定是同一个人
- ✅ **必须**检查作者列表，确认 full name + affiliation 匹配
- ✅ **优先**使用已知合作者（如导师名）交叉确认
- ✅ **搜索范围要广**：不要只找团队内合作论文，要搜全名的独立工作

```bash
# arXiv 搜索
curl -s "https://arxiv.org/search/?query=Jiyan+He&searchtype=author&start=0"

# DBLP 搜索（CS顶会专用，补充arXiv）
curl -s "https://dblp.org/search/publ/api?q=jiyan+he&format=json&h=30"

# Semantic Scholar（备用，国内可用）
curl -s "https://api.semanticscholar.org/graph/v1/author/search?query=Jiyan+He&fields=papers"
```

**常见错误案例：**
- "Haoxiang Guan"（研究生）≠ 广东三角镇政府"[同门A]"
- "Zhang Yanzhi"可能出现不同领域同名者
- 必须通过 affiliation 中的学校/机构判断是不是同一人

### 微信公众号搜索
- 使用 wechat-article-search skill
- **不要放公众号原文链接**（链接会失效）
- **要提取并展示公众号文章的深度内容**（标题 + 核心内容摘要 + 日期）
- 搜到的有意义段落/观点要直接引用到报告正文里

### 学历信息
- 通过论文 affiliation 字段提取
- 如论文中无记录，明确标注"未找到公开信息"，不要猜测
- ❌ 不要根据导师、团队推断某人的本科院校

## 📋 标准工作流程

### 能力1（导师画像）工作流
```
1. 关系核实（最先做！）
   - 确认"导师/师兄/师弟"关系，不要假设
   - 查 arXiv 早期论文 acknowledgement 或 thesis

2. 基本信息收集
   - arXiv + DBLP 作者页搜索（广泛搜索，不局限团队内）
   - 论文 affiliation 提取机构信息
   
3. 实时动态（6个月）
   - wechat-article-search 搜索姓名（中文名）→ 提取内容摘要
   - arXiv 按时间排序，最近6个月论文
   - 百度新闻搜索

4. 历史成果（5年）
   - arXiv + DBLP 完整列表（广泛搜索！不要只找合作论文）
   - 提取代表性论文（影响力/创新性）

5. 深度分析
   - 研究方向演变（时间轴分析）
   - 性格推断（基于公开发言、研究风格）
   - 资源和关系网络（合作者分析）
   - 向上管理建议（具体、可操作）

6. HTML 报告生成 + catbox.moe 上传
```

### 能力2（同门画像）工作流
```
1. 实体消歧（最先做！）
   - 确认这个人在学术圈，不要把同名官员/其他领域人混入
   - 通过学校affiliation、团队合作者确认身份

2. 基本信息
   - 本科/博士院校（arXiv affiliation，或公开简历）
   - 研究方向关键词

3. 论文搜索（广泛！）
   - arXiv + DBLP + Semantic Scholar 多源
   - 不只找和导师合作的论文，找全部论文

4. 实时动态 + 历史成果（同能力1）

5. 个性与合作潜力分析
   - 研究风格（广度/深度型）
   - 技能互补点
   - 性格推断（基于研究风格和公开信息）
   - 共同话题切入点

6. HTML 报告生成 + 上传
```

### 能力3（合作规划）工作流
```
1. 用户输入（必须提前收集）
   - 用户的研究方向/想做的idea
   - 用户的技能背景（语言、框架、数学）
   - 已有实验积累（可选）
   - 首选合作对象（可选）

2. 基于能力1+2的数据
   - 导师最新关注点 → 可以争取导师支持的方向
   - 同门技能图谱 → 互补分工方案

3. 输出（按重要性排序）
   - 人际关系维护策略（最吸引人，放最前）
   - 3-5个具体可行的research ideas（含分工）
   - 综合推进策略与时间线（放最后）
```

## 🎨 HTML 报告规范

### 设计风格
- 卡片：白色背景，`border-radius: 16px`，`box-shadow: 0 8px 32px rgba(...)`
- 时间线：左侧渐变竖线 + 圆点
- 网格：`display: grid; grid-template-columns: repeat(auto-fit, minmax(280px, 1fr))`

### 背景色按报告类型区分
- 能力1（导师）：`linear-gradient(135deg, #667eea 0%, #764ba2 100%)`（紫色）
- 能力2（同门）：`linear-gradient(135deg, #11998e 0%, #38ef7d 100%)`（绿色）
- 能力3（规划）：`linear-gradient(135deg, #f093fb 0%, #f5576c 100%)`（粉红）

3. **研究演变轨迹**（放前面！分阶段时间轴）
4. **深度洞察 · 性格与工作风格**（放前面！）

3. **研究演变轨迹**（放前面！）
4. **深度洞察**（技能图谱、研究风格、性格）
5. **核心合作者网络**（放前面！）
6. **如何与他建立联系**（具体策略）

1. Header
2. **人际关系维护策略**（放最前面！最干货的内容）
3. 合作 Ideas 1-5（分工方案）
4. 综合推进策略与时间线
5. Footer

### 链接规则
- ✅ arXiv链接：`https://arxiv.org/abs/论文ID`，全部可点击
- ✅ DBLP链接：`https://dblp.org/search?q=姓名`，可点击
- ❌ 公众号"查看原文"链接：不要加（链接会失效）
- ✅ 公众号内容：提取文章核心内容/观点直接展示，只保留标题和日期

### 信息标注规范（不能说"核实"）
- 数据来源：标注 `来源：arXiv / DBLP / 百度新闻` 即可
- 推断信息：加`（推断）`标注
- 未找到信息：明确写"未找到公开信息"

## ⚠️ 重要原则

1. **关系必须核实** - "导师/师兄/师弟"这类关系，先查后用，绝不假设
2. **搜索要广泛** - 不要局限于团队内合作论文，要全面搜索
3. **实体消歧** - 同名人物（官员、其他领域）要通过affiliation排除
4. **禁用"核实"等词** - 用"来源：XXX"代替
5. **公众号展示内容** - 不放失效链接，要提取并展示文章内容摘要
6. **章节顺序** - 吸引人的内容（洞察、策略）放前面，论文列表放后面
7. **并行执行** - 多人画像必须用 sub-agent 并行

## 🎨 HTML 报告格式规范（vavrq5.html 标准）

参考文件：`examples/vavrq5.html`

### 关键格式要求
- **单卡片结构**：所有内容在一个 `.card` 中，用 `.section` 分隔，**不要多卡片堆叠**
- **section-title**：必须包含 `<span class="section-icon">` emoji + 文字，带左侧紫色边框
- **highlight-box**：核心建议/行动计划，黄色渐变背景
- **info-box**：数据来源/统计，蓝色渐变背景
- **footer**：白字居中，包含数据来源声明和生成时间
